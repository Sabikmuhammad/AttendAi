# Face Registration

In this project, student identity and face-related fields are managed through student records and image upload flow.

## Main data model fields

Student records include fields such as:

- studentId
- department
- section
- semester
- imageUrl
- faceEmbedding (when available)

## Typical flow

1. Create student from admin UI or API.
2. Upload student image from admin upload endpoint.
3. Run detection/recognition pipeline via AI service.
4. Use attendance marking flow for recognized students.

## Related endpoints

- GET /api/students
- POST /api/students
- GET /api/students/:id
- PATCH /api/students/:id
- DELETE /api/students/:id
- POST /api/admin/upload-student-image
- POST /api/detect-faces

## Recommended practices

- Use clear front-facing photos.
- Ensure one active student identity per studentId per institution.
- Keep institution boundaries strict when testing multi-tenant data.
- Re-upload and refresh embeddings if recognition quality drops.
