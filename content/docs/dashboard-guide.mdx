# Dashboard Guide

## Admin area

Primary pages:

- /admin/dashboard
- /admin/students
- /admin/faculty
- /admin/classes
- /admin/create-class
- /admin/camera
- /admin/profile

Typical admin tasks:

1. Create faculty and student records.
2. Create classes and assign students.
3. Start/stop monitoring and review status.
4. Review attendance records and exports.

## Faculty area

Primary pages:

- /faculty/dashboard
- /faculty/create-class
- /faculty/live-class
- /faculty/profile

Typical faculty tasks:

1. Manage own class sessions.
2. Monitor class activity.
3. Review student attendance results.

## Student area

Primary pages:

- /student/dashboard
- /student/profile

Typical student tasks:

1. Check attendance status.
2. View class-related summaries.

## Data consistency behavior

Class pages and dashboard rely on synchronized status values:

- scheduled
- active
- completed

Status is recomputed from time windows in server-side logic before key class-list responses.
