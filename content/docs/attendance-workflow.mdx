# Attendance Workflow

## End-to-end flow

1. Class is created with schedule and enrolled students.
2. Class status sync updates scheduled/active/completed based on time.
3. Admin starts monitoring from /admin/camera or uses class monitor action.
4. Detection pipeline identifies enrolled students.
5. Attendance is stored through mark endpoint with duplicate prevention.

## Key APIs

- Class listing and creation:
  - GET /api/classes
  - POST /api/classes
- Attendance records:
  - GET /api/attendance
  - POST /api/attendance
- Detection-driven marking:
  - POST /api/attendance/mark
- Monitoring control:
  - POST /api/monitor/start
  - POST /api/monitor/stop
  - GET /api/monitor/status/:classId
  - GET /api/monitor/active

## Duplicate and enrollment checks

Attendance mark flow verifies:

- class exists
- class is active
- student is enrolled in class
- student is not already marked for that class

## Output and reporting

- GET /api/attendance supports filtering by classId, studentId, date range
- CSV export is available by sending format=csv
