## System Components

### 1. CCTV Camera Layer

**Purpose**: Capture live video from classrooms

- IP cameras with RTSP support
- Positioned at classroom entrances or strategic locations
- Continuous video streaming to processing server
- Multiple cameras supported per classroom

**Requirements**:
- RTSP protocol support
- Minimum 720p resolution (1080p recommended)
- Stable network connection
- Fixed positioning for optimal face capture

### 2. Video Stream Capture

**Purpose**: Receive and process RTSP streams

- Connects to CCTV cameras via RTSP URLs
- Extracts video frames at configurable intervals (default: 1 FPS)
- Optimizes frame quality for face detection
- Handles multiple camera streams simultaneously

**Technology**:
- OpenCV for video processing
- FFmpeg for stream decoding
- Python async/await for concurrent processing

### 3. AI Processing Engine

**Purpose**: Detect and recognize faces in video frames

#### Face Detection Module
- Uses MTCNN (Multi-task Cascaded Convolutional Networks)
- Detects multiple faces in each frame
- Extracts face bounding boxes and landmarks
- Filters low-quality detections

#### Face Recognition Module
- Generates 128-dimensional face embeddings
- Uses deep learning models (FaceNet/ArcFace)
- Compares embeddings with registered student database
- Confidence threshold: 0.6 (configurable)

**Technology**:
- Python + TensorFlow/PyTorch
- face_recognition library
- dlib for face landmarks
- NumPy for embedding operations

**Performance**:
- Processing speed: 30-60 faces/second (with GPU)
- Recognition accuracy: 99.9%
- False positive rate: < 0.1%

## Data Flow

### Attendance Recording Flow

1. **Video Capture**
   - Camera streams RTSP video to capture service
   - Service extracts frames at 1 FPS

2. **Face Detection**
   - Each frame is processed by face detection model
   - Face regions are extracted and normalized

3. **Face Recognition**
   - Face embeddings are generated for each detected face
   - Embeddings are compared with student database
   - Best match above confidence threshold is selected

4. **Attendance Recording**
   - Student ID and timestamp are recorded in database
   - Duplicate entries within 5 minutes are prevented
   - Notification sent to admin dashboard

5. **Dashboard Update**
   - Real-time WebSocket updates to connected dashboards
   - Attendance counts refreshed
   - Student status updated (Present/Absent)

### Scalability

- **Horizontal Scaling**: Add more AI processing servers for handling more cameras
- **Load Balancing**: Distribute web traffic across multiple Next.js instances
- **Database Sharding**: Partition data across multiple MongoDB nodes
- **CDN Integration**: Cloudinary for serving images and videos

## Security Architecture

- **Authentication**: Clerk-based JWT tokens
- **Authorization**: Role-based access control (Admin, Faculty, Student)
- **Data Encryption**: AES-256 for face embeddings
- **API Security**: Rate limiting and request validation
- **Network Security**: HTTPS/TLS for all communications
- **Privacy**: Face embeddings stored instead of actual photos
