# Security

## Authentication model

- Login issues access and refresh tokens via HTTP-only cookies.
- /api/auth/me validates the access cookie and returns user context.
- Refresh flow is handled by /api/auth/refresh.

## Authorization and tenancy

- Tenant context is resolved per request.
- Institution scoping is applied to core data queries.
- Role checks are used on privileged management routes.

## Sensitive data and operational guidance

- Do not commit .env.local or secrets.
- URL-encode special characters in MongoDB passwords.
- Keep separate credentials per environment.

## MongoDB safety

- Use Atlas Network Access restrictions.
- Resume paused clusters before testing logins.
- Keep DB users least-privileged for application needs.

## Monitoring and media

- Browser camera mode requires user permission.
- Prefer HTTPS in non-local environments.
- Restrict AI service access to trusted network paths.

## Recommended hardening checklist

- Enable rate limiting on auth-sensitive endpoints.
- Rotate secrets periodically.
- Audit role access to admin and super-admin routes.
- Add request logging and alerting for repeated auth failures.
