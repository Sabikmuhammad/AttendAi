# Security & Privacy

AttendAI is built with security and privacy at its core. This guide explains how we protect your data and ensure system security.

## Data Protection

### Face Embeddings vs Photos

**What We Store**:
- ✅ **Face Embeddings**: 128-dimensional mathematical vectors
- ✅ **Student Metadata**: Name, roll number, email, department
- ✅ **Attendance Records**: Timestamps and confidence scores

**What We DON'T Store**:
- ❌ **Original Photos**: Deleted immediately after processing
- ❌ **Video Recordings**: Frames processed and discarded
- ❌ **Biometric Templates**: Only embeddings stored

### Why Face Embeddings?

Face embeddings provide security advantages:

1. **Irreversibility**: Cannot reconstruct original photo from embedding
2. **Privacy**: No identifiable image data stored
3. **Efficiency**: Fast comparison without image processing
4. **Compliance**: Meets GDPR and privacy regulations

### Encryption

**Data at Rest**:
- Face embeddings: AES-256 encryption
- Database: Encrypted MongoDB Atlas
- Backups: Encrypted storage
- Logs: Encrypted and access-controlled

**Data in Transit**:
- HTTPS/TLS 1.3 for all web traffic
- Encrypted RTSP streams (optional)
- Secure API communication
- Encrypted database connections

## Authentication & Authorization

### User Authentication

**Clerk Authentication**:
- Industry-standard authentication
- Multi-factor authentication (MFA) supported
- Password policies enforced
- Session management
- Social login options

**Password Requirements**:
- Minimum 12 characters
- Uppercase and lowercase letters
- Numbers and special characters
- Not in common password list
- Regular password rotation (90 days)

### Role-Based Access Control (RBAC)

**Admin Role**:
- Full system access
- User management
- System configuration
- All data access

**Faculty Role**:
- View/manage assigned classes only
- Mark attendance for their classes
- Generate reports for their classes
- Cannot access system settings

**Student Role**:
- View personal attendance only
- View enrolled classes
- Generate personal reports
- No access to other students' data

### API Security

**API Token Management**:
- JWT-based authentication
- Token expiration (1 hour default)
- Refresh token mechanism
- Token revocation capability
- API key rotation

**Request Security**:
- Rate limiting (100-1000 req/hour)
- Request validation
- Input sanitization
- SQL injection prevention
- XSS protection

## Network Security

### Firewall Configuration

**Required Ports**:
```
3000 - Next.js web application (HTTPS)
8000 - AI processing service (internal)
27017 - MongoDB (internal/VPN only)
554 - RTSP camera streams (internal network)
```

**Firewall Rules**:
- Block all unnecessary ports
- Internal services not exposed to internet
- Camera network isolated (VLAN)
- VPN access for remote management

### SSL/TLS Configuration

**Production Setup**:
```nginx
server {
    listen 443 ssl http2;
    server_name attendai.com;
    
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    
    # Additional security headers
    add_header Strict-Transport-Security "max-age=31536000" always;
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
}
```

## Privacy Compliance

### GDPR Compliance

AttendAI complies with GDPR requirements:

**Right to Access**:
- Students can view their stored data
- Export personal data in standard format
- API endpoint: `GET /api/students/:id/data-export`

**Right to Deletion**:
- Complete data deletion upon request
- Includes face embeddings and attendance records
- 30-day deletion process
- Confirmation provided

**Right to Rectification**:
- Update incorrect personal information
- Correct erroneous attendance records
- Face re-registration available

**Data Minimization**:
- Only necessary data collected
- No sensitive personal data (race, religion, etc.)
- Automatic data cleanup (configurable retention)

**Consent Management**:
- Explicit consent required before registration
- Consent withdrawal option
- Clear privacy policy
- Parental consent for minors

### FERPA Compliance (US Education)

For US educational institutions:

- Student privacy protected
- Attendance data treated as education records
- Access limited to authorized personnel
- Parent/student access rights respected
- No unauthorized disclosure

## Camera Security

### Camera Network Isolation

**Best Practices**:
1. **Separate VLAN**: Isolate cameras on dedicated network
2. **No Internet Access**: Cameras don't need internet
3. **Firewall Rules**: Block camera-to-camera communication
4. **VPN Access**: Remote access only via VPN

### Camera Authentication

**Secure Configuration**:
- Change default passwords
- Use strong credentials
- Regular password rotation
- Disable unnecessary services (UPnP, Telnet)

**RTSP Security**:
```
# Secure RTSP URL format
rtsp://username:strong-password@camera-ip:554/stream1

# Use RTSPS (RTSP over TLS) if supported
rtsps://username:password@camera-ip:322/stream1
```

### Camera Firmware

**Security Updates**:
- Regular firmware updates
- Patch known vulnerabilities
- Enable auto-update (if available)
- Monitor security advisories
