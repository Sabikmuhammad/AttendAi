# Camera Setup

AttendAI currently supports two monitoring sources from the admin camera page.

## Source modes

1. Browser camera mode
- Uses getUserMedia in the admin browser
- Captures frames and posts them for face detection
- Good for local testing and mobile browser usage

2. AI service mode
- Uses AI monitor endpoints through Next.js proxy routes
- Supports backend monitoring workflows and status tracking

## Required environment variables

In .env.local:

```env
AI_SERVICE_URL=http://localhost:8000
PYTHON_SERVICE_URL=http://localhost:8000
NEXT_PUBLIC_AI_SERVICE_URL=http://localhost:8000
```

## Core routes used by camera flow

- Admin page: /admin/camera
- Start monitor: POST /api/monitor/start
- Stop monitor: POST /api/monitor/stop
- Monitor status: GET /api/monitor/status/:classId
- Active monitors: GET /api/monitor/active
- Face detection upload: POST /api/detect-faces
- Mark attendance: POST /api/attendance/mark

## Browser/mobile camera notes

- Allow camera permissions in the browser.
- Use HTTPS on remote devices when required by browser policy.
- Keep device and classroom lighting stable for better recognition quality.

## AI service health

If monitor start fails, verify AI service is running:

```bash
cd ai-service
python main.py
```

Then test http://localhost:8000/docs.
